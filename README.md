# ReviseIQ

A free Leaving Certificate revision platform for Irish students.  
Past papers · AI quizzes · Topic trend analysis.

---

## Features

- **Past Papers** — Official SEC exam papers filtered by subject, year and level
- **AI Quiz** — 5 custom MCQ questions generated by Claude (Anthropic) on any LC topic
- **Topic Trends** — Bar and line chart analysis of which topics appear most (2015–2024)
- **Pomodoro Timer** — Built-in 25-minute study timer on the homepage
- **Dark Mode** — Toggleable, saved to `localStorage`

## Deploying to Vercel (step by step)

Vercel hosts static sites for free and gives you a public URL accessible from any device.

### Prerequisites

- A [GitHub](https://github.com) account (free)
- A [Vercel](https://vercel.com) account — sign up with your GitHub account (free)

---

### Step 1 — Put the project on GitHub

1. Go to [github.com](https://github.com) and click **New repository**
2. Name it `reviseiq` (or anything you like), set it to **Public**, click **Create repository**
3. Open a terminal in your `LCStudy` folder and run:

```bash
git init
git add .
git commit -m "Initial ReviseIQ commit"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/reviseiq.git
git push -u origin main
```

> **Note:** `config.js` is in `.gitignore` so your API key will **not** be pushed. See Step 3 for how to handle this on Vercel.

---

### Step 2 — Deploy on Vercel

1. Go to [vercel.com](https://vercel.com) and click **Add New → Project**
2. Click **Import** next to your `reviseiq` GitHub repository
3. Vercel will detect it as a static site automatically — **no build settings needed**
4. Click **Deploy**
5. After ~30 seconds, Vercel gives you a live URL like `https://reviseiq.vercel.app`

That URL works on any device — phone, tablet, laptop.

---

### Step 3 — Set your Anthropic API key (for the Quiz page)

Because `config.js` is gitignored, the Quiz page won't work until you set the API key in Vercel:

1. In your Vercel project, go to **Settings → Environment Variables**
2. You **cannot** inject a JS variable directly via Vercel env vars for a static site, so you have two options:

**Option A — Easiest: add config.js before deploying**

Before running `git push`, temporarily remove `config.js` from `.gitignore`, fill in your real key, push once, then re-add it to `.gitignore` and push again. The key will be visible in source — only do this if you are comfortable with that (the Anthropic API key is used client-side anyway, so it is visible in the browser regardless).

```bash
# 1. Edit config.js with your real key
# 2. Temporarily allow it
echo "# config.js" >> .gitignore-bak  # backup
git add config.js
git commit -m "Add API key config"
git push
```

**Option B — Cleaner: Vercel Edge Function (advanced)**

Convert the API call to a Vercel serverless function so the key lives only in Vercel env vars and never reaches the browser. This is a more involved change — see `js/quiz.js` for the fetch call to proxy.

---

### Step 4 — Custom domain (optional)

1. In your Vercel project, go to **Settings → Domains**
2. Add your domain (e.g. `reviseiq.ie`) and follow the DNS instructions
3. Vercel provides a free SSL certificate automatically

## Local development

No build step needed — just open `index.html` in a browser, or use a local server:

```bash
# Using VS Code Live Server extension (recommended)
# Right-click index.html → Open with Live Server

# Or using Python
python -m http.server 3000
# Then open http://localhost:3000
```
